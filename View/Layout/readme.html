AJAX and Highcharts Deployment

I have created 6 charts in 4 different types which covers all charts from the requirement and meets the expectation for "superior work". These types are column, bar, line and bar. I utilize a type of chart specifically according to the data type. For instance, I use bar chart for "Current GPAs of All Students" because we will have hundreds of student and it is better to set students on the y-axis, which is bar chart's advantage.

For all charts, I fetch data from the database with specific queries and let the backend server send the data back in JSON. So, all chart data are dealt with AJAX.

I followed the AJAX and charting examples Jim provides us. I used a select input for users to choose a chart to show. As the user confirm an option. The front page will send a AJAX post request to my server, and then the server fetch data according to the request number and send back in JSON format.

By using AJAX, the DGS don't need to refresh the overview page to load different charts. This enhances user's experience and make DGS's work more efficient.

By using Highchart, the beautiful and useful chart library, the DGS can have very intuitive views upon students' and advisors' statutues. Also, since all charts are set in one overview page, the DGS can have very convenient and direct access to monitor students.

Therefore, the combination of AJAX and Highcharts indeed improves DGS's efficiency and productivity quite decently.
